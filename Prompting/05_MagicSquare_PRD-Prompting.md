**Cursor**

`@Prompting/`에 있는 자료를 먼저 훑어서 요구사항과 범위를 정리한 뒤, 그 내용을 기반으로 바로 사용할 수 있는 PRD 문서 형태로 작성하겠습니다. 관련 파일 구조부터 확인할게요.

`Prompting` 폴더에 4개 원본 프롬프트 문서가 있네요. 핵심 요구사항을 정확히 반영하려고 4개를 모두 읽고 PRD로 통합하겠습니다.

이제 기존 보고서 톤과 구조를 맞추기 위해 현재 `Report` 문서 형식을 짧게 확인한 다음, `PRD`를 새 파일로 작성하겠습니다.

`Prompting`의 4개 문서를 통합해 제품 요구사항 관점으로 재구성한 PRD를 `Report` 폴더에 새 문서로 작성하겠습니다. 핵심은 Invariant 기반 학습 목표, I/O 계약, 사용자 여정, 테스트/품질 기준을 하나로 묶는 것입니다.

`@Prompting/` 내용을 통합해서 PRD를 작성해두었습니다.

- 생성 파일: `Report/05_MagicSquare_PRD.md`
- 반영 범위:
  - 문제정의(표면/개선 정의) 및 핵심 Invariant
  - Epic/KPI, Persona, User Journey, User Stories
  - 입력/출력/오류 계약
  - 기능/비기능 요구사항
  - ECB 아키텍처 제약, TDD/테스트 기준, DoD
  - 리스크 및 대응, 추적성 요구사항

원하면 다음으로 팀 공유용 형식으로 `한 페이지 요약 PRD` 버전도 추가해드릴게요.
