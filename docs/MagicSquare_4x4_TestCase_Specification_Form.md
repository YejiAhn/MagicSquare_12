### 전제 조건 (공통)

- 테스트 대상 함수/클래스가 프로젝트에 존재하거나, 동일 계약의 테스트 더블로 대체 가능할 것.
- 좌표 표기는 구현 계약에 따름(0-기반/1-기반 통일). 본 명세의 좌표 예는 **0-기반 (행, 열)** 가정으로 서술했으며, 구현과 다르면 해당 TC의 입력·기대만 치환하면 됨.

### 특별하게 필요한 절차 (공통)

- 우선순위 **P1** TC는 회귀·스모크에 포함 권장.
- 제품 요구가 “빈칸 정확히 2개”인 경우, **A-003~A-005**·**C-003** 등은 *보조 함수 단위* 또는 *검증기 이전 단계*에서의 동작 명세로 해석하고, 최종 `solution()` 입구에서는 도메인 규칙에 맞게 거부할 수 있음.
